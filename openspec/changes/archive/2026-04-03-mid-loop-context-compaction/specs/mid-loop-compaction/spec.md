## ADDED Requirements

### Requirement: State compaction triggers on size threshold
The system SHALL compact the LangGraph state inside `agent_node` before calling the LLM when either of the following conditions is true: total content character count across all messages exceeds `compaction_char_threshold` (default: 40,000), OR total message count exceeds `compaction_message_threshold` (default: 30). Both conditions are evaluated on every `agent_node` entry except during a cooldown period after a recent compaction.

#### Scenario: Compaction fires on char threshold
- **WHEN** the state contains messages whose total content length exceeds 40,000 characters
- **THEN** `maybe_compact_state()` is called before the LLM invocation, the state is compressed, and a `context_compacted` trace event is emitted

#### Scenario: Compaction fires on message count threshold
- **WHEN** the state contains more than 30 messages regardless of total char count
- **THEN** `maybe_compact_state()` is called before the LLM invocation and the state is compressed

#### Scenario: Cooldown prevents consecutive compactions
- **WHEN** a compaction was performed fewer than `cooldown_messages` (default: 10) messages ago
- **THEN** compaction is skipped even if thresholds are exceeded

#### Scenario: Compaction skipped when no summary LLM is configured
- **WHEN** `summary_llm` is `None` on the graph instance
- **THEN** `maybe_compact_state()` returns the state unchanged and no error is raised

### Requirement: Compaction preserves critical messages
The system SHALL never evict the following messages during mid-loop compaction: the first message in the state (the user's original task), the most recent `retain_recent` messages (default: 8), and any `ToolMessage` with `status="error"`.

#### Scenario: User task message is preserved
- **WHEN** mid-loop compaction runs on a state with 50 messages
- **THEN** the first message in the original state remains as the first message in the compacted state

#### Scenario: Recent messages are preserved
- **WHEN** mid-loop compaction runs
- **THEN** the last 8 messages (by position) are retained verbatim in the compacted state

#### Scenario: Error tool messages are preserved
- **WHEN** the state contains a `ToolMessage` with `status="error"` outside the recent window
- **THEN** that message is retained in the compacted state, not summarized

### Requirement: Evicted messages are replaced by a summary block
The system SHALL replace evicted `AIMessage` and `ToolMessage` entries with a single `assistant` message prefixed with `[Context Summary]:` containing a concise summary of: tools called, repos analyzed, key findings, decisions made, and errors encountered.

#### Scenario: Summary replaces old tool results
- **WHEN** compaction evicts 20 messages including directory_tree and file read results
- **THEN** a single `[Context Summary]:` message replaces them, and the state message count is reduced

#### Scenario: Summary is generated by the configured summary LLM
- **WHEN** compaction runs and `summary_llm` is set to kimi_reader
- **THEN** kimi_reader's `ainvoke()` is called to generate the summary, not the mode's reasoning LLM

#### Scenario: Compaction failure is non-fatal
- **WHEN** the summary LLM call raises an exception
- **THEN** the original state is returned unchanged, the exception is logged as a warning, and agent execution continues normally
